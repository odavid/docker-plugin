Prerequisites: Docker image must derive from jenkins/jnlp-slave and Jenkins master has to be accessible over network from container.

If checked, the container command will pass the jenkins slave connection parameters and will not inject the slave jar. This option enables the user to perform some manipulation within the entrypoint before starting the slave. For example, start the container with root and use gosu to go back to jenkins user. This pattern let the docker slave run docker builds. See https://github.com/odavid/jenkins-jnlp-slave

The following command will be passed: 

-url <MASTER_URL> [-tunnel <TUNNEL>] <SECRET> <AGENT_NAME>

For more details, see jenkins/jnlp-slave
